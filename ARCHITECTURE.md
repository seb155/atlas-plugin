# ATLAS Plugin Architecture

> Concise reference. Deep dive: `.blueprint/ARCHITECTURE.md`

## Plugin Scopes (6 Domains)

| Domain | Skills | Focus |
|--------|--------|-------|
| **core** | 20 | Memory, session, context, vault, profile, doctor. Required by all others. |
| **dev** | 39 | TDD, planning, debugging, code review, git, CI, shipping. |
| **frontend** | 5 | UI design, browser automation, visual QA, product health. |
| **infra** | 11 | VM/container ops, deploy, network audit, mesh diagnostics, security. |
| **enterprise** | 14 | Governance, knowledge engine, agent teams, programme management. |
| **experiential** | -- | Episodes, relationships, intuitions (experimental). |

Legacy 3-tier mode (`user` -> `dev` -> `admin`) still builds via `./build.sh all`.

## Skill Routing

```
User prompt --> atlas-assist (master skill, generated per tier/domain)
                   |
                   +--> Matches intent to sub-skill via category + description
                   |    (e.g., "write tests" -> tdd, "deploy" -> devops-deploy)
                   |
                   +--> Invokes sub-skill with full context
```

`atlas-assist` is auto-generated by `scripts/generate-master-skill.sh` at build time.
It reads the resolved skill list, groups by category, and produces a routable SKILL.md.

## Hook System

**Registry**: `hooks/hooks.json` (SSoT -- never put hooks in `settings.json`)

| Event | Examples | Sync/Async |
|-------|----------|------------|
| `SessionStart` | session-start, inject-datetime, statusline-sync | mixed |
| `UserPromptSubmit` | timestamp-injector, prompt-intelligence, theory-of-mind | async |
| `PostToolUse` | post-edit-lint, frontend-ux-gate, code-quality-check | async |
| `PreCompact` | pre-compact-context | sync |
| `Stop` | cleanup-worktrees, session-end-memory-cleanup | async |

**Handler rules**:
- Scripts in `hooks/` with NO `.sh` extension (except `run-hook.sh` dispatcher)
- Must be executable (`chmod +x`)
- Branded output: `ATLAS | {emoji} {CATEGORY} | {message}`
- Async handlers: max 5s timeout, exit 0 on error

**Dispatcher**: `run-hook.sh` routes named sub-hooks (e.g., `run-hook.sh inject-datetime`).

## Build Pipeline

```
1. Read VERSION file
2. For each tier/domain:
   a. resolve_field() -- walk YAML inheritance chain recursively
   b. Copy resolved skills, agents, commands, refs
   c. Copy ALL hooks (wildcard -- every tier gets every hook)
   d. Generate atlas-assist SKILL.md (dynamic counts + persona)
   e. Generate plugin.json + marketplace.json
3. Output: dist/atlas-{tier}/ (self-contained, no runtime deps)
```

**Install**: `make dev` runs `scripts/dev-install.sh` which builds + copies to `~/.claude/plugins/cache/`.
**Publish**: `make publish-patch` bumps VERSION, builds, tests, tags, and pushes (CI creates release).

## CLI: atlas-cli.sh

The shell launcher (`scripts/atlas-cli.sh`, 38 lines) loads 7 modules from `scripts/atlas-modules/`:

| Module | Purpose |
|--------|---------|
| `launcher.sh` | Core atlas() function, tmux integration |
| `dispatch.sh` | Route subcommands to skills/agents |
| `subcommands.sh` | Register atlas subcommands |
| `completions.sh` | Tab completion for zsh/bash |
| `platform.sh` | OS/platform detection |
| `ui.sh` | Terminal UI helpers (colors, spinners) |
| `topics.sh` | Help topic registry |

Installed via `dev-install.sh` which syncs the launcher to the user's shell profile.
