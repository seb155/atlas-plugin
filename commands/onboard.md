# /onboard — Team Readiness Check

Assess whether the project is ready for a new team member. 12 checks across docs, dev environment, CI, and team support. Grades A-F.

**Usage**: `/atlas onboard [subcommand]`

Invoke Skill 'onboarding-check'.

ARGUMENTS: $ARGUMENTS

Subcommands:
- `/atlas onboard` — Full 12-dimension scorecard with grade
- `/atlas onboard gaps` — Actionable list of missing items to fix (priority order)
- `/atlas onboard fix` — Auto-generate missing docs/configs with HITL approval
- `/atlas onboard simulate` — Simulate fresh clone → first commit path
